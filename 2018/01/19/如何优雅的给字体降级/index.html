如何优雅的给字体降级

2018-01-19

最近写一个内嵌于微信中的页面, 用到了weui样式库。因特殊需求, 需要在PC端上做预览, 大多数浏览器是正常的, 但是在IE下却会报如下错误

考虑到是weui的字体所造成的问题, 决定去weui的官方repo下issue中寻求帮助, 结果发现在相关的IE兼容性问题中, weui的维护者表明目前暂时不考虑对IE的兼容。维护人原话如下：

weui主要是针对移动端页面而开发的，IE可能暂时不做考虑

那么只好自己想办法解决啦。

首先利用谷歌和overstackflow, 我们可以知道导致CSS3114出现的原因是字体的权限不足, 在IE中被拦截了, 在stackoverflow的相关问题下有很多看起来不错的答案, 但是和我们遇到的问题都有点差别, 最明显的就是问题下答案都是替换字体文件, 或者增加一份字体文件备份, 多引入一份字体用于IE。 而在weui中, 其字体的引入是直接通过内联的base64字符串而不是外部字体文件引用。

虽然有点不一致, 但是理一理思路, 解决方案也很简单, 大致是以下四步。

  • (1)提取weui中指代字体的base64源字符串
    将上文中的

去除前面的src: url('data:application/octet-stream;base64,, 以及后面的') format('truetype');。
然后得到

这就是我们所需要的base64源字符串了

  • (2)将(1)中的base64字符串转换为字体文件
    接着, 我们通过字体转换网站, 将我们前面一步得到的base64, 转换为字体文件。将我们的base64字符串复制到这个网站的主输入框中, 然后依次勾选decode the data from a Base64 string (base64 decoding) 和 export to a binary file, filename: base64.bin后, 点击Convert to source data按钮, 便可以得到一个名为base64.bin的转换后的字体文件了。

  • (3)通过工具修改字体文件的permission
    接着, 我们通过Make TTF font embeded这个网站, 修改我们字体文件的permission, 这里的字体文件是我们上一步操作中得到的base64.bin。

  • (4)将修改permisision后的字体文件再转换为base64字符串,并替换源weui中的base64字符串
    最后, 我们再次通过字体转换网站, 将上一步中得到的字体文件转换回base64字符串。 先选择我们修改permission后的字体文件, 然后依次勾选encode the source data to a Base64 string (base64 encoding)和output to a textbox (as a string), 接着点击Convert to source data按钮后, 其第二个主输入框中展现出来的内容, 就是我们所需要的转换后的base64字符串了, 我们将其复制, 直接替换我们weui文件中原本的base64就好了。需要说明的是, 因为我们主要使用场景依旧是移动端, 需要考虑资源大小的影响, base64字符串(8kb)较大, 所以这里使用了替换, 而不是俩行并行, 增添一行备份。且后续修改后进行了一系列测试, 并没有问题, 俩行并行的意义不大。
